## Conflict Resolution Notes

### somewm_api.c — `some_set_seat_keyboard_focus()`
**Took upstream version** (commit `25ce8f6`). Upstream went through 3 iterations:
1. `5761b81` — our original cherry-pick
2. `df28bf2` — removed `client_activate_surface()` (broke Firefox/Ghostty popups)
3. `25ce8f6` — final: activate only for X11, early return for XDG

**Preserved:** `some_update_pointer_constraint(surface)` call at end — needed for game mouse lock on NVIDIA.

### somewm.c — `apply_geometry_to_wlroots()`
**Preserved our `xdg->initialized` guard** (commit `5a28e42`). Upstream does NOT have this. Prevents crash during output hotplug when `closemon() -> setmon() -> resize()` reaches uninitialized XDG surfaces. Critical for NVIDIA where DRM timing differs.

### somewm.c — `updatemons()`
**Took upstream version** which adds `updatemons_pending` deferred re-execution. This builds on our original `in_updatemons` guard and handles scale-change reentrancy.

### objects/client.c — property registration
**Took upstream refactored version** using `luaA_class_add_properties()` batch API instead of individual calls. Functionally identical, includes our `aspect_ratio` property.

### Debug logging
**Dropped** all `[FOCUS]`, `[FOCUS-API]`, `[MAPNOTIFY-FOCUS]` debug markers per upstream cleanup. **Kept** `[SOMEWM-DEBUG]` startup markers (fork-only, useful for NVIDIA debugging).
